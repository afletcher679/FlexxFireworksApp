# Android SSL / Supabase Troubleshooting

## Symptoms

```
javax.net.ssl.SSLHandshakeException:
java.security.cert.CertPathValidatorException:
Trust anchor for certification path not found.
```
…on Supabase `fetch` (login, fetching tables, etc.), and/or a build failure:
```
AAPT: error: resource raw/lets_encrypt_root ... not found.
```

## Root causes

1. **Build failure** — a `network_security_config.xml` referenced a `res/raw`
   certificate file that did not exist. (Fixed by the config plugin in this
   repo, which now actually creates the file.)
2. **SSL trust error** — Supabase uses a Let's Encrypt cert chaining to
   **ISRG Root X1**. That root is trusted by default on **Android 7.1+ (API 25+)**
   but NOT on older devices/emulators. Testing on an old emulator (API <= 24)
   triggers the "Trust anchor not found" error.

> ⚠️ This is a managed / CNG Expo project: the `android/` folder is generated by
> `expo prebuild` and is gitignored. **Never hand-edit `android/`** — changes are
> wiped. All native config must go through an Expo **config plugin**.

---

## Recommended fix (clean, no cert bundling)

1. Delete any broken native edits and regenerate:
   ```bash
   rm -rf android ios
   npx expo prebuild --clean
   ```
2. Use a modern emulator/device: **API 33+ (Android 13+)**.
   Android Studio → Device Manager → Create Device → pick an API 33+ image.
3. Rebuild. The SSL error is gone because modern Android trusts Let's Encrypt.

If you only ever target modern devices, you can REMOVE the
`./plugins/withLetsEncryptNetworkSecurity` entry from `app.json` — you don't
need it.

---

## Fix to also support OLD devices (Android <= 7.0)

This repo includes an Expo config plugin that bundles the ISRG Root X1 root CA
and trusts both the system store and that cert.

1. Make sure the cert exists (already committed; re-download if needed):
   ```bash
   bash scripts/download-isrg-cert.sh
   ```
2. Ensure `app.json` has the plugin (already added):
   ```json
   "plugins": [
     "./plugins/withLetsEncryptNetworkSecurity",
     ...
   ]
   ```
3. Regenerate native code and build:
   ```bash
   npx expo prebuild --clean
   npx expo run:android
   ```

The plugin will:
- copy `android-assets/isrg_root_x1.pem` → `android/app/src/main/res/raw/isrg_root_x1.pem`
- write `android/app/src/main/res/xml/network_security_config.xml`
- set `android:networkSecurityConfig` in the manifest

## Other things to check

- **Trailing slash:** `supabaseUrl` must have **no** trailing slash
  (`https://xxxx.supabase.co`, not `.../`). Already fixed in `src/lib/supabase.ts`.
- **Corporate proxy / VPN / Charles / mitmproxy:** any TLS-intercepting proxy
  will also produce "Trust anchor not found". Disable it or install its cert.
- **Device clock:** a wildly wrong system date makes certs look invalid.
- **Confirm device API level:** `adb shell getprop ro.build.version.sdk`
  (24 or below = old trust store; this is the usual culprit).
